Fix ubuntu login loop

  • Why happened?
    Zoom을 설치 후, 한글 입력이 되질 않아서 지금 쓰고 있는 uim(Byeoru)한글 입력기 대신
    fcitx한글 입력기로 변경하려고 함. fcitx설치 후 재부팅 도중에 로그인 후 화면 멈춤현상 발생.

  • How to Fix it
    tty에 들어가서
    현재 사용하고 있는 gdm3말고 다른 diplay manager LightDM과 KDE(sddm)으로 변경

    다른 display manager로 변경해도 해결x

Xauthority 제거

Xauthoriry는 user home디렉토리에 저장되어 있는 파일로 xauth에 대한 쿠키 파일로 xsession에 대한
인증 파일로 사용되는 파일이라고 한다. 처음에 Xauthority가 존재하지 않아서 시도를 안했는데,
lightdm과 sddm설치하면서 새로 생긴 것 같다.

이 방법도 해결x

gdm3 재설치
방금 전에 설치한 LitghtDM과 KDE를 삭제하고, gdm3를 다시 설치해봤다.

gdm3로 변경할 때 아래의 오류메세지가 나올 수 있는데 그냥 reboot하면 gdm3가 적용된다.

마지막으로,

TTY(Teletypewriter) - OS에서 제공하는 가상콘솔
데스크탑 GUI(xwindows)에서는 /dev/pts/1에 올려져 있고
tty 콘솔 모드(Ctrl+Alt+F1F6)에서는 `/dev/tty16`
tty가 물리적인 콘솔에서 사람이 직접 조작하는 개념이라면, pty는 xwindow에서 에뮬레이터로 접속할
때 사용하는 접속점의 개념
로그인을 콘솔에서 했는지, 원격지에서 했는지 차이
